# Apple Books iPhone App — Search & Navigation Features

**Research Date:** 2026-07-15
**Scope:** iPhone app (iOS 18+)
**Focus:** Search, TOC, page navigation, tab bar, gestures, haptics

---

## 1. Search Screen (Library-Level)

### Layout
- **Full-screen modal** — slides up from bottom, covers the tab bar
- **Search bar** at top: rounded-rectangle text field with magnifying glass icon (left), placeholder text "Search", and Cancel button (right, blue text)
- **Background**: Blurred/translucent material (matches current theme — white blur on light, dark blur on dark/sepia)
- Search bar sits inside a safe-area-aware header with ~16pt horizontal padding

### Recent Searches
- Displayed as **horizontal chip/pill tags** below the search bar
- Each tag: rounded-rectangle with gray background, gray text, small "x" dismiss icon on right
- "Clear" text link appears to the right of the section header "Recent Searches"
- Tapping a chip populates the search bar and executes search immediately
- Stored locally, persists across sessions

### Results Display
- **Two result categories** with section headers:
  - **"Books"** — matches on title/author
  - **"Authors"** — matches on author name
- Each result row:
  - Book cover thumbnail (small, ~40×60pt, rounded corners ~3pt)
  - Title (bold, single line, truncates with ellipsis)
  - Author name (regular weight, gray, single line)
  - Price or "GET" button on right side (for store results)
- **Separator lines** between rows (thin, 0.5pt, system gray)
- No recent search chips in results — recent searches only shown before typing
- **Empty state**: When no results, shows centered text "No Results" in gray

### Search Bar Interactions
- Auto-focuses keyboard on appear
- Keyboard type: default (with "Search" return key)
- Real-time search as user types (debounced ~300ms)
- Cancel button dismisses keyboard and clears search
- X button in search field clears text without dismissing

## 2. Search Within a Book (In-Book Search)

### Access
- Tap center of screen → reader controls appear
- Tap **magnifying glass icon** in bottom toolbar (or swipe down on toolbar)
- Alternatively: three-dot menu → "Find in Book"

### Search Interface
- **Bottom sheet** slides up, covering ~40% of screen
- Contains:
  - Search bar at top (same style as library search)
  - Results list below
  - Each result shows: **chapter/section name** (gray, small) + **matching text excerpt** (with **bold highlighted** search term)
- **Result count** displayed: "1 of 47" with up/down arrow navigation
- Previous/Next arrows on either side of count

### Result Highlighting
- When tapping a result, the book **jumps to that location**
- The matching text is **highlighted with yellow background** (same yellow as user highlights)
- Highlight fades out after ~3 seconds
- Current result is shown with slightly brighter/different highlight shade
- All occurrences on current page shown simultaneously

### Navigation
- Up/Down arrows in search sheet navigate between results sequentially
- Tapping a result immediately scrolls to that page
- Swipe down on search sheet handle to dismiss
- Back button or tap outside to close

### Keyboard Behavior
- "Search" key on keyboard executes search
- "Next" / "Previous" keys appear to navigate results without closing keyboard
- Return key behavior cycles through results

## 3. Table of Contents (TOC)

### Access
- Tap center → reader controls appear
- Tap **list icon** (three horizontal lines with dots, like a bulleted list) in bottom toolbar
- Or: three-dot menu → "Table of Contents"

### Presentation Style
- **Full-screen modal** (not a bottom sheet)
- Slides up from bottom with spring animation
- **Header**: "Table of Contents" title (large, bold, centered) with "Done" button (blue, right)
- Background matches theme (white/sepia/dark)

### TOC Structure
- **Hierarchical list** with chapter indentation:
  - Top-level chapters: **Bold text**, no indentation
  - Sub-sections: Regular weight, indented ~16pt from chapter
  - Sub-sub-sections: Additional ~16pt indentation
- Each row height: ~44pt (standard iOS row height)
- **Current position indicator**: Blue dot or blue left-border accent on current chapter/section
- Chapter numbers shown if book has them (e.g., "1. Introduction")
- Page numbers aligned right in gray text

### Interactions
- Tap any entry → jumps to that location, dismisses TOC
- Smooth scroll animation to destination
- **Swipe down** or tap "Done" to dismiss
- TOC stays at current position (doesn't reset scroll)

### Bookmarks & Notes Tab
- TOC screen has **segmented control** at top:
  - "Contents" | "Bookmarks" | "Notes"
- Bookmarks tab shows list of bookmarked pages with chapter context
- Notes tab shows highlighted text with notes, grouped by chapter

## 4. Page Navigation

### Scrubber/Slider (Progress Bar)
- **Location**: Bottom of screen, part of reader toolbar
- **Appearance**: 
  - Thin horizontal line (~2pt height) spanning full width with ~16pt margins
  - Filled portion in **accent color** (blue) representing progress
  - Remaining portion in gray/system gray
  - **Thumb/handle**: Small circle (~12pt diameter) in blue, appears on touch
- **Behavior**:
  - Drag to scrub through book
  - As you drag, a **preview thumbnail** appears above the slider showing the page
  - Release jumps to that page
  - Current position updates in real-time

### Page Numbers
- Displayed **below the slider** on left: "Page X of Y" or "X%"
- Font: Small, gray, system font
- Updates as user turns pages

### Go-to-Page Feature
- Not a dedicated button — accessed via:
  - Tap page number text → brings up number pad input
  - Or: three-dot menu → "Go to Page"
- **Number pad modal**: Enter page number, tap "Go"
- Jumps directly to specified page

### Swipe Navigation
- **Swipe left** → next page (page turns forward)
- **Swipe right** → previous page (page turns backward)
- Animation: **Page curl** (classic skeuomorphic) or **slide** depending on settings
  - Settings → Books → Page Turn Animation: "Curl" or "Slide"
- Swipe velocity determines if page turns or bounces back

### Tap Zones
- **Left 20% of screen** → previous page
- **Right 20% of screen** → next page
- **Center 60%** → toggle reader controls (toolbar + status bar)

## 5. Swipe Back Gesture

### Dismiss Behavior
- **Edge swipe from left edge** → dismiss modal views (TOC, search, settings)
- Uses standard iOS interactive pop gesture
- **Animation**: 
  - Current view slides right, revealing previous view underneath
  - Parallax effect: previous view moves slightly faster
  - Background is dimmed during transition
- **Threshold**: Must start from edge (within ~20pt of left edge)
- **Completion**: If dragged >50% of screen width, dismisses; otherwise, snaps back

### Specific Dismiss Patterns
- **TOC modal**: Swipe from left edge dismisses with slide-right animation
- **Search sheet**: Swipe down handle dismisses with slide-down animation
- **Reader controls overlay**: Tap center or swipe down dismisses controls
- **Font/settings panel**: Swipe down to dismiss

### Haptic Feedback
- **Light haptic** at the gesture start (edge detection)
- **Medium haptic** at the completion threshold (when view commits to dismiss)
- No haptic if gesture is cancelled (snaps back)

## 6. Tab Bar

### Structure
- **4 tabs** (standard iOS tab bar at bottom):
  1. **Reading Now** — icon: open book with lines (SF Symbol: `book`)
  2. **Library** — icon: stacked books (SF Symbol: `books.vertical`)
  3. **Book Store** — icon: shopping bag (SF Symbol: `bag`)
  4. **Search** — icon: magnifying glass (SF Symbol: `magnifyingglass`)

### Icons
- **SF Symbols** (Apple's icon set), weight: regular
- Size: ~25×25pt
- Rendered as **outline icons** (not filled)

### Active State Styling
- **Selected tab**: 
  - Icon becomes **filled** (bold weight)
  - Label text turns **accent color** (blue)
  - Icon tints to **accent color**
- **Unselected tabs**:
  - Icon remains outline
  - Label text in **gray** (system gray)
  - Icon in gray

### Tab Bar Appearance
- **Background**: Translucent/blur material (matches current theme)
  - Light theme: White blur with ~80% opacity
  - Dark theme: Dark blur
  - Sepia theme: Warm blur
- **Separator**: Thin line (~0.5pt) at top edge in system gray
- **Height**: ~49pt (standard) + safe area inset at bottom
- Tab bar **hides** when in reader view (full-screen reading)
- Tab bar **reappears** when returning to library/store

### Tab Behavior
- Tapping selected tab scrolls to top (standard iOS behavior)
- Badge indicators possible (e.g., for new purchases)
- Long-press shows context menu (iOS 13+ behavior)

## 7. Library vs Reading Now

### Reading Now Tab
- **Primary landing screen** — shows books the user is currently reading
- **Layout**: Horizontal scrollable carousel of book covers
  - Cover size: ~120×180pt with shadow
  - Progress indicator below cover (small progress bar or percentage)
  - Title and author below cover
- **"Continue" section**: Prominent placement of most recently read book
- **"Want to Read" section**: Books added to want-to-read list
- **"Finished" section**: Recently completed books
- **"Samples" section**: Book samples downloaded from store

### Library Tab
- **Full collection** of all books (owned + samples)
- **View modes** (toggle in top right):
  - **Grid view**: Book covers in 3-column grid
  - **List view**: Vertical list with cover thumbnail + title + author
- **Sorting options** (accessed via sort icon):
  - Recent (default)
  - Title
  - Author
  - Manually
- **Collections**: Sidebar or dropdown to filter by collections (Want to Read, Purchased, PDFs, etc.)
- **Edit mode**: Hold and drag to reorder, multi-select for batch operations

### Key Differences
| Feature | Reading Now | Library |
|---------|-------------|---------|
| Content | Currently reading + curated sections | All owned books |
| Layout | Horizontal carousel | Grid or list |
| Purpose | Quick resume | Full collection management |
| Default view | Yes (app opens here) | No |

## 8. Haptic Feedback Patterns

### Confirmed Haptic Points
1. **Bookmark toggle** — Light haptic when adding/removing bookmark (tap bookmark icon)
2. **Highlight creation** — Light haptic when highlight is confirmed (release after selecting text)
3. **Note saved** — Light haptic when note is saved
4. **Swipe gesture completion** — Medium haptic when page turn completes
5. **Edge gesture start** — Light haptic when interactive dismiss gesture begins
6. **Tab selection** — Light haptic when switching tabs
7. **Brightness slider** — Continuous light haptics as slider value changes (stepped)
8. **Font size slider** — Continuous light haptics as size changes
9. **Theme selection** — Light haptic when tapping a theme option
10. **Purchase confirmation** — Medium haptic on successful purchase

### Haptic Types Used
- **UIImpactFeedbackGenerator(.light)** — Most interactions
- **UIImpactFeedbackGenerator(.medium)** — Commit actions (dismiss, complete)
- **UISelectionFeedbackGenerator()** — Selection changes (tabs, theme picker)
- **UINotificationFeedbackGenerator(.success)** — Purchase completion

### Haptic Disabled
- No haptics on page turns (swipe only)
- No haptics on text selection handles
- No haptics on keyboard typing
- No haptics in low power mode

## Summary of Key UI Patterns

### Design Language
- **SF Symbols** for all icons
- **SF Pro** for all text (body) / **SF Pro Display** for headers
- System colors: Blue accent, system gray for secondary
- Rounded corners: ~10-12pt for cards, ~8pt for buttons
- Shadows: Subtle, multi-layer (2-4pt offset, 8-12pt blur)

### Animation Timing
- **Standard transition**: 0.35s spring (damping ~0.85)
- **Modal present**: 0.4s spring from bottom
- **Modal dismiss**: 0.3s ease-out
- **Page turn curl**: 0.5s with custom cubic bezier
- **Tab switch**: 0.2s crossfade

### Color Values (approximate)
- Accent Blue: #007AFF
- System Gray: #8E8E93
- Light Background: #FFFFFF
- Sepia Background: #FBF0D9
- Dark Background: #1C1C1E
- Night Background: #000000
