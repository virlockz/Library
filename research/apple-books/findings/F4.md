# Apple Books iPhone App — Theme System & Visual Design Tokens

> Research compiled from Apple HIG, developer forums, CSS epub theme detection, community-measured values, and iOS design system documentation.

---

## 1. Reading Theme Colors (ePub Content Area)

Apple Books provides **4 reading themes** for the book content area. These are the **exact hex values** obtained from screenshots (originally measured on iPhone 6, confirmed consistent through iOS 16+):

| Theme | Background Color | Text Color | CSS Selector (epub) |
|-------|-----------------|------------|---------------------|
| **White (Light)** | `#fbfbfb` | `#000000` | `:root[__ibooks_internal_theme*="White"]` |
| **Sepia** | `#f8f1e3` | `#000000` | `:root[__ibooks_internal_theme*="Sepia"]` |
| **Gray** | `#5a5a5c` | `#c9caca` | `:root[__ibooks_internal_theme*="Gray"]` |
| **Night** | `#121212` | `#b0b0b0` | `:root[__ibooks_internal_theme*="Night"]` |

### Notes on Themes
- The "White" theme is slightly off-white (`#fbfbfb`), not pure `#ffffff`
- The "Gray" theme is a medium-dark warm gray; text is light gray
- The "Night" theme forces `background-color: black !important` on `:root` and overrides all child elements' backgrounds to `transparent !important` with text color forced to `#bebebe !important`
- In Night mode, Apple applies this CSS override internally:
  ```css
  :root { background-color: black !important; }
  :root * { background-color: transparent !important; color: #bebebe !important; }
  ```
- **Focus Theme** (iOS 16+): Apple introduced a "Focus" theme with background approximately `#FFFCF4` (measured on iPhone 13 Pro Max, iOS 16.5) — a warmer, slightly yellowish white. Exact value may vary by device/OS.

### Theme Detection (CSS for epub authors)
```css
:root[__ibooks_internal_theme*="Sepia"] { /* sepia overrides */ }
:root[__ibooks_internal_theme*="Gray"]  { /* gray overrides */ }
:root[__ibooks_internal_theme*="Night"] { /* night overrides */ }
```
- On iOS 15+, class names changed to lowercase: `epub-sepia`, `epub-white`, `epub-gray`, `epub-night`
- On macOS: `BKSepiaStyleTheme`, etc.
- iOS 16+: `__ibooks_internal_theme` attribute may return `undefined` — use `@media (prefers-color-scheme: dark)` as fallback
- For dark theme custom text: add class `ibooks-dark-theme-use-custom-text-color` to `<html>`

## 7. Tab Bar

### iOS 25 and Earlier (Pre-iOS 26)
- **Style**: Full-width bar at bottom of screen
- **Background**: Translucent with blur effect (`UIBlurEffect(style: .systemChromeMaterial)`)
- **Translucency**: `isTranslucent = true` by default
- **Border**: Thin 0.5pt hairline separator at top (separator color varies by theme)
- **Blur type**: System material blur (dynamic, adapts to background content)

### iOS 26+ (Liquid Glass)
- **Style**: Floating, centered, **pill-shaped** capsule (NOT full-width)
- **Inset**: 21pt from left, right, and bottom edges
- **Background**: Liquid Glass material — translucent, light-refractive glass texture
- **Search**: Separate circular Liquid Glass "island" to the right of tabs
- **Behavior**: Minimizes/scrolls away when user scrolls down; expands when scrolling up
- **Labels**: 11pt SF Pro, regular weight
- **Selected state**: Brand/accent color for icon + label
- **Unselected state**: `#1A1A1A` (light) / `#F5F5F5` (dark)

### Tab Bar Color Values
| Property | Light Mode | Dark Mode |
|----------|-----------|-----------|
| Background | System material blur | System material blur (dark) |
| Unselected text | `#1A1A1A` | `#F5F5F5` |
| Selected text | `#007AFF` (accent) | `#0A84FF` (accent) |
| Separator | `#3C3C43` @ 29% | `#545458` @ 39% |

## 8. Navigation Bar

- **Default**: Translucent with blur (`isTranslucent = true`)
- **Blur effect**: `UIBlurEffect(style: .systemChromeMaterial)` — the standard iOS system material
- **Translucency level**: System-managed; content scrolls behind the bar
- **Title behavior**: Large title (34pt bold) transitions to small title (17pt semibold) on scroll
- **Scroll edge effect**: Content fades out + blurs as it approaches the bar (fade extends to bottom of nav bar)
- **Background**: Clear/transparent — blur effect provides the visual background
- **iOS 26+**: Adopts Liquid Glass material; floats above content

### Navigation Bar Color Values
| Property | Value |
|----------|-------|
| Background | Transparent (blur provides visual) |
| Title text (light) | `#000000` |
| Title text (dark) | `#FFFFFF` |
| Large title size | 34pt bold |
| Small title size | 17pt semibold |
| Bar tint | System default |

## 10. Glass / Blur Effects Throughout the App

### System Blur Materials (iOS 14+)
| Material | Use Case | Opacity/Intensity |
|----------|----------|-------------------|
| `.systemChromeMaterial` | Nav bars, tab bars, toolbars | Standard system blur |
| `.systemThinMaterial` | Lightweight overlays | Thinner blur |
| `.systemThickMaterial` | Heavy overlays | Thicker blur |
| `.systemUltraThinMaterial` | Very subtle overlays | Minimal blur |
| `.systemMaterial` | General purpose | Medium blur |

### Liquid Glass (iOS 26+)
- **Description**: A layered, translucent visual style where UI surfaces feel fluid and responsive to motion
- **Components**: Nav bars, tab bars, toolbars, buttons, menus, floating controls
- **Properties**:
  - **Lensing**: Bends and concentrates light in real-time (vs. traditional blur that scatters light)
  - **Materialization**: Elements appear by gradually modulating light bending
  - **Fluidity**: Gel-like flexibility with instant touch responsiveness
  - **Morphing**: Dynamic transformation between control states
  - **Adaptivity**: Multi-layer composition adjusting to content, color scheme, and size
- **Where used**: Navigation layer only — NOT on content layers like lists
- **Tab bar specific**: Floating pill-shaped glass container, content fades out below it

### Blur Implementation Notes
- Navigation bars: `isTranslucent = true` + system blur effect
- Tab bars: `configureWithDefaultBackground()` adds blur + translucency
- Toolbars: Same as nav bars
- Custom glass effect in SwiftUI: `.glassEffect()` modifier
- Custom glass effect in UIKit: `UIVisualEffectView` with `UIBlurEffect`

## 11. Typography

### Primary Fonts
| Font | Use Case |
|------|----------|
| **SF Pro** | System font for all UI text — body, labels, navigation |
| **New York** | Serif companion font — used in Apple Books for book content, Apple News |
| **SF Pro Rounded** | Used in some contexts for softer UI elements |

### SF Pro Details
- **Classification**: Neutral, flexible, sans-serif
- **Weights**: 9 weights (Ultralight, Thin, Light, Regular, Medium, Semibold, Bold, Heavy, Black)
- **Variants**: SF Pro Text (small sizes), SF Pro Display (large sizes)
- **Widths**: 4 widths available
- **Rounded variant**: SF Pro Rounded available
- **System font**: Used via `systemFont(ofSize:)` or `.font(.system)` in SwiftUI

### New York Details
- **Classification**: Transitional serif typeface
- **Introduced**: 2019
- **Use in Books**: Appears prominently in Apple Books for book content rendering
- **Design**: Inspired by classic serif designs with modern screen rendering
- **Features**: Generous x-heights, carefully tuned stroke contrast

## 12. Icon Style

### SF Symbols
- **System**: Apple's native icon system — SF Symbols
- **Style**: Both outlined and filled variants available
- **Default**: Outlined variant is the "universal" / default
- **Filled variant**: Used for emphasis, selection states, tab bar selected items
- **Weights**: 9 weights matching SF Pro (Ultralight through Black)
- **Scales**: 3 scales (Small, Medium, Large)
- **Tab bar icons**: Typically use `medium` scale, `regular` weight
- **Selected state**: Filled variant + accent color
- **Unselected state**: Outlined variant + gray/secondary color
- **Alignment**: Automatically aligns with SF Pro text at all weights and sizes

### Icon Sizes (Approximate)
| Context | Size |
|---------|------|
| Tab bar icon | 25x25pt |
| Navigation bar button | 22x22pt |
| List item icon | 29x29pt (larger) or 22x22pt |
| Settings icon | 22x22pt |

## 14. CSS `prefers-color-scheme` Support

Apple Books (iOS 12+) supports the standard CSS media query:
```css
@media (prefers-color-scheme: dark) {
  /* Dark theme styles */
}
@media (prefers-color-scheme: light) {
  /* Light theme styles */
}
```

This differentiates between Light/Sepia (light) and Gray/Night (dark) as a binary switch.

## 15. Key Design Principles

1. **Clarity**: Text is legible, icons are precise
2. **Deference**: UI helps users interact with content but never competes with it
3. **Depth**: Provides sense of context and hierarchy
4. **Content-first**: Navigation layer (glass/blur) floats above; content is always primary
5. **Adaptive**: All colors, blur effects, and materials adapt to light/dark mode automatically
